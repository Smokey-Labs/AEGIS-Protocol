# Tool Authority Contract

**Status:** Beta  
**Owner:** Gregory “Smokey” McEver  
**Last Updated:** 2026-01-08

---

## Purpose

Defines which tools may ever be invoked and under what evidence conditions.

---

## Authority Surface

Prevents Aurora from performing unauthorized actions.

---

## Inputs

- Tool metadata
- Evidence requirements
- Authority Envelope

---

## Outputs

- Tool eligibility matrix

---

## Failure Modes

- Unauthorized tool exposure  
- Implicit tool permission

---

## DriftWatch Integration

Any tool invocation without contract alignment emits DriftWatch.

---

## Non-Goals

- Tool logic execution  
- Prompt interpretation
